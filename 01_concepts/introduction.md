# 쿠버네티스(Kubernetes) 소개

## 1. 쿠버네티스란?
쿠버네티스(Kubernetes, 줄여서 K8s)는 컨테이너화된 애플리케이션을 자동으로 배포, 스케일링 및 관리해주는 오픈소스 오케스트레이션 플랫폼입니다. 구글에서 내부적으로 사용하던 보그(Borg) 시스템을 기반으로 탄생했습니다.

### Docker Compose vs Kubernetes
- **Docker Compose**: 단일 호스트(머신)에서 여러 컨테이너를 실행하고 관리하는 데 적합합니다.
- **Kubernetes**: 여러 대의 서버(클러스터)를 하나의 시스템처럼 관리하며, 고가용성과 확장성이 필요한 운영 환경에 적합합니다. (Docker Compose의 확장판 개념)

## 2. 주요 장점 상세

### 🚀 배포 및 관리 자동화
- 새 버전을 배포할 때 점진적으로 업데이트(Rolling Update)하거나, 문제가 생기면 이전 버전으로 되돌리는(Rollback) 기능을 제공합니다.

### ⚖️ 부하 분산 (Load Balancing)
- 서비스에 트래픽이 몰릴 때 여러 개의 Pod로 부하를 자동으로 나누어 줍니다. 내장된 서비스 디스커버리 기능을 통해 컨테이너 간 통신이 원활하게 이루어집니다.

### 📈 자동 스케일링 (Scaling)
- CPU 사용량 등이 설정값 이상으로 올라가면 컨테이너 수를 자동으로 늘리고(Scale-out), 한가할 때는 줄여서(Scale-in) 리소스를 효율적으로 사용합니다.

### 🛠 셀프 힐링 (Self-healing)
- 컨테이너가 종료되거나 응답이 없는 경우, 쿠버네티스는 이를 감지하고 설정된 상태(Desired State)를 유지하기 위해 컨테이너를 자동으로 다시 시작하거나 교체합니다.
